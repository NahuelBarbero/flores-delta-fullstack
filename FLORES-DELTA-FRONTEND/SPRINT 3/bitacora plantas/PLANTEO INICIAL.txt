2. Informacion Primitiva del Cliente (Chat de WhatsApp)

Detalles:
El sistema esta disenado para administrar integralmente un growshop y su cultivo interno, desde el catalogo de productos hasta la trazabilidad de cada planta dentro de los espacios de cultivo.

El modelo ERD define la estructura de datos base que el backend y el frontend deben implementar para garantizar la coherencia, trazabilidad y escalabilidad del sistema.

Componentes principales:
- Administrador (ADMINISTRADOR): Representa al usuario con rol de gestion. Puede crear y editar productos, registrar plantas, anadir notas y registrar riegos. Es la figura que interactua con todos los modulos de la aplicacion.
- Catalogo de productos (PRODUCTO): Contiene los productos que el growshop ofrece (semillas, fertilizantes, insumos, etc.). Cada producto tiene un ID_inventario unico, precio, stock y descripcion. Permite construir el catalogo publico visible en la web.
- Espacios de cultivo (ESPACIO + MAPA_POSICION): El cultivo se organiza en tres espacios principales, cada uno con posiciones o "slots" dentro de un mapa (MAPA_POSICION). Cada planta se ubica en una posicion dentro de un espacio. Este modelo permite un mapa visual (tipo grid) para mostrar donde esta cada planta.
- Plantas (PLANTA): Es la entidad nucleo del sistema de cultivo. Cada planta tiene: Su genetica (GENETICA), Su forma de cultivo (FORMA_CULTIVO), Su espacio y posicion (ESPACIO / MAPA_POSICION), Fechas de siembra, Estado (archivada o activa), Notas resumen.
- Etapas de cultivo (ETAPA_PLANTA + REGISTRO_ETAPA_PLANTA): Define las fases de crecimiento de cada planta: germinacion, crecimiento, floracion, etc. Cada registro guarda fechas de inicio y fin, observaciones y valores medidos (temperatura, humedad, pH, etc.). Permite analizar el historico de desarrollo y comparar plantas o geneticas.
- Bitacora (NOTA + REGISTRO_RIEGO + IMAGEN_PLANTA): Registra todas las interacciones y observaciones del administrador: NOTAS: comentarios y observaciones con fecha. REGISTROS DE RIEGO: volumen, estado y fecha. IMAGENES: fotos asociadas a cada planta. Esto construye una trazabilidad completa del ciclo de vida de cada planta.

Logica General:
- El Administrador es el actor central.
- Las Plantas se agrupan en Espacios, ubicadas en posiciones del mapa.
- Cada Planta tiene una genetica, una forma de cultivo, y pasa por varias etapas con registros especificos.
- Los Productos representan el catalogo del negocio (visible al publico).
- Las Notas, Riegos e Imagenes conforman la bitacora personal del cultivo.

El modelo esta preparado para ser persistido en una base de datos relacional (PostgreSQL o MySQL) y consumido via API REST o GraphQL por el frontend en React.
